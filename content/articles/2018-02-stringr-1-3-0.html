We are happy to announce that stringr 1.3.0 is now on CRAN. stringr provides a cohesive set of functions designed to make working with strings as easy as possible. For a complete list of changes, please see the release notes.

What’s new?

  • Two new wrappers from the glue package: str_glue() and str_glue_data().

  • str_flatten(), a wrapper for the stri_flatten() function, which flattens a character vector into a single string.

  • str_remove() and str_remove_all(), which wrap str_replace() and str_replace_all() for removing patterns from strings.

  • str_squish() removes whitespace from the left and right sides of strings, and converts multiple spaces or space-like characters from the middle of strings into a single space.

str_squish("\n\nString \t\nwith all \tthis     \rspace   in it.\n\n")
#> [1] "String with all this space in it."
  • str_sub() now has an omit_na argument which can be used to ignore NAs and keep the original strings.
x1 <- x2 <- x3 <- x4 <- "AAA"
str_sub(x1, 1, NA) <- "B"
str_sub(x2, 1, 2) <- NA
str_sub(x3, 1, NA, omit_na = TRUE) <- "B"
str_sub(x4, 1, 2, omit_na = TRUE) <- NA
c(x1, x2, x3, x4)
#> [1] NA    NA    "AAA" "AAA"

API changes

The long deprecated str_join(), ignore.case(), and perl() functions have been removed. So, you may see Error : object ‘ignore.case’ is not exported by 'namespace:stringr' during package build.
